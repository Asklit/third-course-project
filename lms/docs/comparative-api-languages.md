## 2. Фреймворки Python для backend API

| Характеристика | FastAPI | Django REST Framework | Flask | aiohttp |
|---|---|---|---|---|
| Архитектурный подход | API-first, современный ASGI-фреймворк | Полноценная экосистема Django + REST-слой | Минималистичный микрофреймворк | Асинхронный фреймворк с более низким уровнем абстракции |
| Поддержка асинхронности | Встроенная | Ограниченная, исторически ориентирован на sync-сценарии | Через расширения и дополнительную настройку | Встроенная |
| Автогенерация OpenAPI и Swagger | Встроенная | Поддерживается через schema-инструменты | Обычно через внешние библиотеки | Чаще требует дополнительной настройки |
| Валидация данных | Встроенная, на основе Pydantic | Serializer-классы | Обычно вручную или через расширения | Обычно вручную или через внешние модели |
| Производительность | Высокая | Средняя | Средняя | Высокая |
| Скорость разработки API | Очень высокая | Высокая, особенно в крупных Django-проектах | Высокая для небольших сервисов | Средняя |
| Порог входа | Низкий-средний | Средний | Низкий | Средний |
| Удобство для микросервисов | Высокое | Среднее | Высокое | Высокое |
| Подходящие сценарии | REST API, микросервисы, сервисы с документацией из коробки | Админ-панель, ORM, крупные web-системы | Небольшие API и прототипы | Асинхронные сервисы с ручным контролем логики |

Такое сравнение позволяет аргументированно прийти к выбору `FastAPI` для вашего backend-сервиса.
