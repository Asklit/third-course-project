## Текущий контракт сдачи

Актуальный формат `POST /assignments/{assignment_id}/submit`:

- `report_file` - один `.docx`, опционально
- `code_files[]` - файлы кода, опционально
- `submission_meta` - JSON-строка

`submission_meta`:

```json
{
  "assignment_id": 1,
  "comment": "Первая версия",
  "submitted_at": "2026-03-17T12:00:00Z",
  "code_mode": "file",
  "code_link": "",
  "delete_report": false,
  "delete_code": false
}
```

Поддерживаемые сценарии:

- первая сдача отчета и кода;
- только отчет;
- только код;
- замена одной части;
- удаление одной части;
- код как ссылка на репозиторий.

## Что уже достаточно хорошо для демонстрации

- единая точка входа через `core`;
- разделение на микросервисы;
- отдельное хранилище учебных материалов;
- файловая сдача и хранение метаданных;
- реальный поиск по wiki;
- фронтенд, который показывает полный пользовательский сценарий.

## Что стоит доделать в первую очередь

### Высокий приоритет

- [ ] recovery flow: `forgot-password` / `reset-password`
- [ ] история попыток сдачи как отдельный endpoint или отдельная таблица версий
- [ ] автоматические тесты для `core` и `wiki`
- [ ] формальный OpenAPI / API examples в документации

### Средний приоритет

- [ ] роли `teacher` / `assistant`
- [ ] выставление оценки и комментария преподавателя
- [ ] отдельный audit trail действий
- [ ] более формализованный retry/outbox для callback

### Низкий приоритет

- [ ] S3/MinIO вместо локального файлового хранения
- [ ] email/telegram уведомления
- [ ] observability: Prometheus/Grafana
- [ ] CI/CD

## Что особенно хорошо смотрится для курсовой

- `wiki` как отдельный микросервис с собственным поиском;
- `Meilisearch` как внутренний search engine;
- `curated`-слой материалов плюс пайплайн конвертации;
- частичная сдача и обновление только одной части работы;
- связка `postgres + mongo + filesystem + meilisearch` с четким разделением ответственности.

## Что важно не забыть к защите

- показать сценарий: логин -> список работ -> переход в ЛР -> поиск -> сдача отчета -> дозагрузка кода;
- показать, где именно лежат файлы и метаданные;
- показать, что `core` не хранит wiki-логику, а только проксирует и агрегирует;
- показать, что поиск работает даже при отказе `Meilisearch` через fallback.
